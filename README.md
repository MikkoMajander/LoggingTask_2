I started by creating the folder layout as instructed. I created the src folder, and inside it I created the four required files: main.js, routes.js, counter.js, and logger.js.


Before writing any code, I initialized the Node.js project and installed the required dependencies:
 
npm init -y
 
npm install express@4.18.2 winston@3.11.0
 

First, I implemented the logger. I reused the same Winston configuration from Task 1.

With the logger ready, I updated main.js to use it and to delegate routing.

In routes.js, I created the three required endpoints:

GET /counter-increase

GET /counter-read

GET /counter-reset

I also added logging for each request using the endpoint syntax.

Next, I implemented the in-memory counter in counter.js. The module maintains a single integer and exposes the three methods: read, increase, and reset.

I also added the counter log messages as required.

I tested each method manually by calling them in Node REPL before wiring them into the routes.


Once everything was connected, I started the server with the command:

node src/main.js


I used the rest.http file to send requests:

GET http://localhost:3000/counter-increase
GET http://localhost:3000/counter-read
GET http://localhost:3000/counter-reset

Each request produced three layers of logs:

main logs when the server starts

endpoint logs when a route is hit

counter logs showing the internal counter state
